Ordinary Least Squares (OLS)¶

Despite what your professors told you about the Church of OLS.

OLS is actually a very specific formula that turns the regression problem into what's known in math as close-formed solution

$\beta=(X'X)^{-1}X'Y$

This is the actual formula, you essentially invert the matrix and can solve for the "global minimum"

End of story.....right!¶

In our work this isn't very computationally expensive as the # of variables in most of our models are extremely reasonable for computational purposes. But if you had 1,000+ variables in your model the computation would begin to become much more complex. In these instances gradient descent is necessary.

Remember when people talk about the importance of standardization or normalization?¶

The truth is that when it comes to OLS it's irrelevant, the formula is just as easy to solve if the numbers are range for one variable is 500k as they are if they are if the range is 3. Because you aren't calculating derivatives it doesn't matter. But if you are using gradient descent those numbers make a huge differenece.

In this article we'll first compare Gradient Descent and Ordinary Least Squares on the Boston Housing Dataset and then we will generate an enormous dataset and compare them again.¶

In Numpy you will need to use indexing, I'll show you how it works quickly, so we can see what the means and standard deviations look like after standardizing.

  2. Second we provide brackets after the X
  3. Third we can place 2 values inside the brackets, because our matrix is 2-d. The first will identify the row and the second the column.

Now we are going to instantiate the class by adding the input and output dimensions. If you remember the shapes we looked at above we can get from the second column in the 2-day arrays shapes we looked at earlier. Which happen to be 12 and 1.

learning_rate = 0.01

In [19]:
epochs = 500
for epoch in range(epochs):
    epoch += 1
    # Convert numpy array to torch Variable
    inputs = torch.from_numpy(X_standard).requires_grad_()
    labels = torch.from_numpy(y_standard)
    # Clear gradients w.r.t. parameters
    optimizer.zero_grad() 
    # Forward to get output
    outputs = model(inputs)
    # Calculate Loss
    loss = criterion(outputs, labels)
    # Getting gradients w.r.t. parameters
    loss.backward()
    # Updating parameters
    optimizer.step()
print('epoch {}, loss {}'.format(epoch, loss.item()))

Gradient Descent¶

Let's just rewrite everything from above to ensure nothing has changed. Notice the input_dim is now 5000.

In [26]:
# Create class
class LinearRegressionModel(nn.Module):
    def __init__(self, input_dim, output_dim):
        super(LinearRegressionModel, self).__init__()
        self.linear = nn.Linear(input_dim, output_dim)  
    
    def forward(self, x):
        out = self.linear(x)
        return out

input_dim = 5000
output_dim = 1

model = LinearRegressionModel(input_dim, output_dim)

criterion = nn.MSELoss()
learning_rate = 0.01
optimizer = torch.optim.SGD(model.parameters(), lr=learning_rate,momentum=0.9)

epochs = 150 # 1000 epochs again
def torch_gradient_descent():
    t1 = time.time()
    for epoch in range(epochs):
        epoch += 1
        # Convert numpy array to torch Variable
        inputs = torch.from_numpy(X_standard).requires_grad_()
        labels = torch.from_numpy(y_standard)
        # Clear gradients w.r.t. parameters
        optimizer.zero_grad() 
        # Forward to get output
        outputs = model(inputs)
        # Calculate Loss
        loss = criterion(outputs, labels)
        # Getting gradients w.r.t. parameters
        loss.backward()
        # Updating parameters
        optimizer.step()
    print(time.time()-t1)
    print('epoch {}, loss {}'.format(epoch, loss.item()))
